# 数据节点

数据节点是微应用的数据存储，数据是根据账号和微应用id进行存储，每个微应用可以有多个数据节点，每个数据节点可以多个key对象，具体使用方式参阅[API](api#data_node)。数据是储存在数据表中的，根据账号同步的


## 权限说明

数据范围`scope`分为`app`和`user`, 分别对应`APP`和`USER`。数据范围支持设置为公共数据。公共数据在公开模式下可以**读取**。非公共数据仅在各账号模式下可以**读写**。

- APP级别的数据，供所有账号全局使用。
- 用户级别的数据，供当前登录账号使用、账号之间数据隔离。


权限示例具体如下：
|登录模式/范围（scope）|APP `public`|APP |USER `public`|USER |
|---|---|---|---|---|
|公开模式|读|-|读|-|
|普通账号登录|读|-|读、写|读、写|
|管理账号登录|读、写|读、写|读、写|读、写|

举例说明：
一个天气的微应用
- 城市列表数据（普通用户只能查看和选择）-> APP `isPublic=true`
- 用户的选择的城市（普通用户可以修改）-> USER `isPublic=true`
- 天气API接口调用所需的账号、密码、Token（管理账号可以修改）-> APP
- 普通账号自配置API Token信息 -> USER
- 普通账号自配置城市列表 -> USER `isPublic=true`

如果无法确定数据范围，请完全使用 USER 级别的数据节点。
